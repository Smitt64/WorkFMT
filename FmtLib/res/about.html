1.0.1.1

  • Исправлена ошибка, приводящая к удалению индекса при попытке исключить сегмент из соседнего индекса;
  • Исправлена ошибка, не позволяющая изменять сегментам индекса свойство «Комментарий»;
  • Добавлено автоматическое создание полей «T_FMTBLOBDATA_XXXX» и «T_FMTBLOBDATA_XXXX» для таблиц с переменной частью;
  • Теперь после удаления уникального индекса, список доступных ключей для свойства «Первичный ключ» обновляется корректно. Если индекс указан в качестве первичного ключа, то при его удалении соответствующее свойство таблицы будет установлено значение -1;
  • При изменении размера полей с типом STRING, CHR, UCHR и SNR будет автоматически заполняться свойство «Знаков после запятой». Свойство будет сброшено при изменении типа поля на отличное от строкового;
  • В функцию «Создать из текста» добавлено распознание типа поля по строкам «NUMBER(10)» и «NUMBER(5)»;
  • В окне сообщений добавлена возможность фильтровать записи по типу «Сообщения», «Предупреждения» и «Ошибки»;
  • Доработаны интерфейсные ограничения на ввод наименований и комментариев к таблице, полям и индексам;
  • При изменении свойства «Имя таблицы» введенная строка будет приводиться к нижнему регистру, а так же отсекаться лидирующие и завершающие пробелы;
  • На панели редактирования, в списке полей при наведении курсора мыши на кнопку «удалить» будет помечаться красной рамкой запись, которая будет удалена. При наведении курсора мыши на кнопку «добавить» синей линией будет указано место, в которое будет вставлено новое поле;
  • Команды «Fmt->XML» и «TableSql» сгруппированы в меню «Действия», и теперь называются «Экспорт в XML» и «Сохранить CreateTablesSql скрипт» соответственно;
  • В меню «Действия» формы редактирования записи Fmt словаря добавлена команда «Перестроить смещения»;
  • При изменении свойства «Временная» таблицы будет меняться иконка формы редактирования;
  • Добавлена автоматическая проверка корректности наименования поля на форме редактирования записи таблицы;

1.0.0.3

  • Исправлена ошибка, не дававшая удалить подключение к базе из списка ранее использованных;
  • Исправлено падение при нажатии кнопки «Закрыть» в пустом списке открытых окон;
  • На стандартной панели инструментов для команды "Новое подключение" добавлен выпадающий список, содержащий историю соединений, как на форме "Авторизация Oracle";
  • В панели редактирования записи Fmt словаря добавлена команда, позволяющая скопировать в буфер обмена строковые заготовки на основе наименования таблицы ("PARTY", "DPARTY_DBT", "dparty_dbt", "FilePARTY", "PARTY_KEYS", "FindPARTY", "iOpenPARTY", "SKF_PARTY");
  • Измен SQL тип поля «UCHR» с «NUMBER(3)» на «RAW»;
  • Добавлена возможность объединять таблицы в группы для более быстрой фильтрации:
  • В меню «Сервис» добавлен пункт «Редактировать группы», запускающий панель создания/редактирования групп и привязки к ним таблиц;
  • В команду «Фильтр» перечня «Таблицы» добавлен выпадающий список с командами «Фильтр», применяющую сортировку к списку, «Показать из группы», применяющую сортировку по всем таблицам, относящимся к выбранной группе, «Добавить в группу», команда, добавляющая таблицы из текущего списка к существующей группе, «Создать группу», команда, создающая новую группу из таблиц текущего списка, «Из много строчного текста», команда, создающая фильтрацию из таблиц, содержащихся в тексте.

1.0.0.2

  • Исправлена ошибка, не дававшая изменить тип поля с «INT» на любое другое;
  • Фильтрация перечня таблиц по нескольким наименованиям;
  • При добавлении нового поля в конец таблицы строка ввода новой записи будет оставаться в области видимости;
  • Команды «Импорт из xml файлов», «Импорт xml из каталога» и «Экспорт в xml файл» добавлены на стандартную панель инструментов;
  • Добавлено цветовое обозначение подключений к базе данных. В окне создания/редактирования соответствующим цветом помечается, к какому подключению относится данная панель;
  • Добавлено меню «Вид», в котором можно показать/скрыть панели инструментов;
  • Добавлены меню и панель инструментов «Окно». На панели инструментов теперь можно быстро переключаться между формами редактирования;
  • Добавлена возможность создавать скрипты для «СreateTableSql».
